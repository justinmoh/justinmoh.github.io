## Development Environments: Essential Setup

### Update Operating System and Install Xcode

After a clean install of our OS X, we install the pre-aquired updates:
* osxupdatecombo10.10.{#}.dmg
* secupd2015-{###}yosemite.dmg

As the time of writing:
* the latest `osxupdatecombo` is `10.10.5`
* and there are also `secupd2015` version `004` and `006` in additional

We can totally skip these manual updates and let `AppStore` to run for itself. Doing this way just save us some times to download stuffs, as we have many other apps to download.

The first thing after we complete the updates, we, install `Xcode`. At the time of writing, the latest version is `Xcode 7.2`. It can be installed using AppStore, or we can just install it with our `.dmg` installer we downloaded earlier.


### System Preferences

While the network is busy with downloads, we take a look at our system settings. There are stuffs that we would likely to set them right in `System Preferences`:

* Mission Control
	* Keyboard and Mouse Shortcuts ( *collide with IDE shortcut key in later stage* )
		* Mission Control: `F9`
		* Application windows: `F10`
* Language & Region
	* Preferred languages:
		* *Add any preferred language*
		* Use `Advanced...` button to change/modify regional format to your preference
* Security & Privacy
	* General
		* Require password `immediately` after sleep
* Keyboard
	* Input Sources
		* *Add any input source as you preferred and set them right*
	* Shortcuts ( *they conflict with each others in default settings* )
		* Spotlight
			* `Restore Defaults` (Should be `Ctrl Space` and `Ctrl Option Space`)
		* Input Sources
			* Tick both of the boxes and make sure their shortcuts are `Cmd Space` and `Option Cmd Space`
* Trackpad
	* Point & Click
		* `Tap to click`
* iCloud
	* *Setup your iCloud account!*
	* Sync everything (except Photos as optinal)
* Internet Accounts
	* *Add Google, Facebook and other accounts that can be used by Mac OS X*

#### ~/.bash_profile

Update (or create) `${HOME}/.bash_profile`:

```bash
#! /usr/bin/env bash
echo '
# let `ls` to show color
alias ls="ls -G"

# format terminal prompt
export PS1="\[$(tput bold)\]+[\[$(tput sgr0)\]\W\[$(tput bold)\]]: \u\[$(tput sgr0)\]\$ "
' >> "${HOME}/.bash_profile" && source "${HOME}/.bash_profile"
```


### Package Manager


#### [Homebrew](http://brew.sh)

`Homebrew` is a package manager for Mac OS X. It can be used to install packages, for example, `php` and `mysql`. To get `homebrew`:

```bash
#! /usr/bin/env bash
ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"
```
After homebrew is installed, you can use `brew doctor` to check if `homebrew` is healthy. For the first time, you may see a `warning`:

> Warning: Homebrew's sbin was not found in your PATH but you have installed formulae that put executables in /usr/local/sbin.  
> Consider setting the PATH for example like so  
> 		echo 'export PATH="/usr/local/sbin:$PATH"' >> ~/.bash_profile

Well then just add update our PATH variable:

```bash
#! /usr/bin/env bash
echo '
export PATH="/usr/local/sbin:$PATH"
' >> ~/.bash_profile && source ~/.bash_profile
```
`Homebrew Doctor` may also warns about `Your Xcode is outdated`. That is because `homebrew` depends on `Command Line Tools` which is bundled with `Xcode` installation -- in fact, many other Development Apps also depend on it.


#### [Homebrew Caskroom](http://caskroom.io)

There is also a `homebrew caskroom`, that, provide a *shortcut* to download apps, like an `AppStore` in Terminal. To get `caskroom`:

```bash
#! /usr/bin/env bash
brew tap caskroom/cask
```


#### *[Github](https://github.com)

**Github** is not directly related to this *package manager*, but as the *ground of open-source projects*, **Homebrew** sends request to `github api` very often. `Github API rate limit` is limiting the access limit from an IP, in a certain timeframe. To get higher rate limit, we need to be authenticated by `github` -- create an access token with your own github account.

So, if you have never done this yet, follow the steps:

1. Create an github account.
2. Generate a [`New personal access token`](https://github.com/settings/tokens/new?scopes=&description=Homebrew)
3. Copy your personal token, a long string like:
> a0u0japo7joujgx3grgjmhfvipz0p4t6xpuyhh0r

4. And put this into your `~/.bash_profile` as homebrew's environment variable:
```bash
#! /usr/bin/env bash
echo '
export HOMEBREW_GITHUB_API_TOKEN=a0u0japo7joujgx3grgjmhfvipz0p4t6xpuyhh0r
' >> "${HOME}/.bash_profile" && source "${HOME}/.bash_profile"
```
